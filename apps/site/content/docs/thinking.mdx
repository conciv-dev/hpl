## Prompts are source, code is artifact

The `.napl` file is the program. It says what the code must _do_ — public API, behavior, edge cases — and leaves every implementation choice to the agent. Generated code is a build artifact: you read it, run it, ship it, never edit it. To change behavior, you change the prompt.

The toolchain enforces this. `napl gen <target>` runs five stages, and nothing is locked until the connection between prose and code is proven:

1. **Snapshot** — hash prompts, snapshot the target tree.
2. **Agent** — a `claude` coding agent works inside `.napl/src/<target>/`, free to scaffold and edit many files.
3. **Test gate** — the target's test command runs (`vitest run`, `cargo test`, …). Failures feed back to the agent, up to three retries.
4. **Attribution gate** — the model maps prompt lines to the changed code lines. A hard gate: three failed attempts leave the files unlocked, the module flagged `unattributed`, and `napl status` at exit 1. Code without a traceable prompt is never green.
5. **Lock** — files go read-only (`0444`), hashes are recorded, one journal line is appended for [prompt blame](/docs/cli/blame).

Gen is incremental: only changed prompts regenerate, with minimal edits to owned regions.

## Drift is a compile error

Edit a locked file anyway and you have **drift**: `napl gen` refuses to run, `napl status` exits 1, CI fails. A compile error you can't act on is a wall, so drift ships with a guided report — the drifted files, a diff against the recorded baseline, and the three ways out:

1. **Fold the edit into the prompt** — `napl reconcile <target> --module <m>` turns your edit into a proposed prompt amendment; the next gen must reproduce it, proven by tests.
2. **Discard the edit** — `napl gen <target> --module <m> --force`. The prompt wins.
3. **Say what you meant** — edit the prompt, then `napl gen <target>`.

The friction sits at compile and commit time, not edit time: locks are `0444`, editors offer one-click "make writable", dev servers never care. The guard rails are asymmetric on purpose — a human gets the quick unlock, while AI agents are mechanically denied by the deny rules and file banners `napl init` installs.

## The machine answers back

Gen doesn't only emit code. The model writes into the margin of your prompt — a `.mapl` file per module, regenerated on every compile:

- **ambiguity** — your words were unclear. A red squiggle, with a suggested rewording.
- **assumption** — a decision the prose left open. A warning.
- **no-op** — the prompt changed but the code didn't; the entry must say why, or the gen fails.
- **note** — hover-visible reasoning about why the code is shaped this way.

Ambiguity becomes a compile diagnostic. The voices have spellings: `.napl` (or `.🧑`) is yours, `.mapl` (or `.🤖`) is the reply — alias rules in [Prompt files](/docs/format#emoji-aliases).

## Modules, not paths

A module's identity is its frontmatter `module` name, never its file path. Discovery walks the tree, so prompt files move freely; two files claiming one module name is a hard error. Moved generated files heal git-style: identical content at a new path is re-recorded as a move and relocked, mostly-similar content heals as moved-plus-drifted, an ambiguous match refuses to guess.
