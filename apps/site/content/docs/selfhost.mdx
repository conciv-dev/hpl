---
title: Self-host
description: NAPL regenerating its own Rust toolchain from prompts, proven module by module against the hand-written test corpus.
---

The endgame: rewrite NAPL's own toolchain as `.napl` prompts targeting Rust, with the hand-written binary acting as **stage0** — the compiler that generates its successor.

```
selfhost/*.napl              behavior prose + tests as given/expect data
        │  stage0: napl gen rust
        ▼
selfhost/.napl/src/rust/     real crates the agent wrote, locked after cargo test
        │
        ▼
equivalence gates            the hand-written modules' own unit-test corpora,
                             replayed against the generated code
```

The fixpoint is **behavioral, not byte-identical**: a generated module passes when the hand-written module's exact test corpus passes against it, whatever the implementation looks like. The pilot module (`body_lines`, frontmatter line-offset math) genned on attempt 1, passed its corpus 5/5, and flagged the one thing the prose left open — the return type — as a machine-layer assumption rather than hiding it.

Not every module gens cleanly. Scaling keeps an **escape-hatch list**: modules that stay hand-written until their prompt drives a generation that passes its corpus. The list is the honest measure of how much of the toolchain self-hosts.
