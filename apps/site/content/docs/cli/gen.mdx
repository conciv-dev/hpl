---
title: napl gen
description: Run a coding agent that writes target code from prompts, then lock and derive IR and attribution.
---

Runs the [gen loop](/docs/thinking) — snapshot, agent, test gate, attribution gate, lock — for each prompt whose content hash changed for the target. Blocks first on [drift](/docs/thinking#drift-is-a-compile-error).

```text title="usage"
Usage: napl gen [OPTIONS] <TARGET>

Arguments:
  <TARGET>  Target language (e.g. typescript, react)

Options:
  -f, --force            Regenerate every prompt even when the prompt has not changed
      --full             Force from-scratch generation instead of automatic incremental mode
  -m, --module <MODULE>  Scope the run to a single module by name
  -h, --help             Print help
```

| Flag | Effect |
| --- | --- |
| `-f, --force` | Regenerate every prompt regardless of changes. Also the way to discard a drifted edit — the prompt wins. |
| `--full` | From-scratch generation instead of automatic incremental mode. |
| `-m, --module` | Scope the run to one module. |

```bash
napl gen typescript --module greeting
```

Gen is incremental by default: the agent receives only the prompt diff and the attribution entries for changed lines. A full scaffold happens only when no prior gen exists.
